## Task 2: Full PCI recursive bus scan

**Objective:** Enumerate all PCI buses, not just bus 0. Detect PCI-PCI bridges
and scan secondary buses recursively. Required for devices behind bridges.

**Files:**
- Modify: `src/drivers/pci.c`

**Approach:**
1. Add a `pci_scan_bus(bus)` function that iterates slots 0-31, functions 0-7
2. For each device, check if it's a PCI-PCI bridge (class 0x06, subclass 0x04)
3. If bridge found, read secondary bus number and call `pci_scan_bus()` on it
4. Call `pci_scan_bus(0)` from `pci_init()`

**Verification:** `pci_list` shows devices on all buses, not just bus 0.

## Task 3: Fix dmesg command buffer size

**Objective:** cmd_dmesg uses a 16KB static buffer but the ring buffer is 64KB.
When the ring has wrapped, 16KB is not enough to read it all.

**Files:**
- Modify: `src/shell/cmds/cmd_dmesg.c`

**Fix:** Replace static buffer with a loop that reads the ring in chunks:

```c
char chunk[512];
int total = 0;
int pos = 0;
// Read in a loop until we've exhausted the ring buffer
// or use kprintf_dmesg_flush_serial() equivalent
```

Simpler: increase static buffer to match DMESG_BUF_SIZE (64KB). With 128KB
kernel stacks, 64KB on the stack is still fine for the shell command context.
Or use a heap allocation.

**Verification:** `dmesg` in shell prints all buffered output without truncation.

## Task 5: Reliable ccache caching in CI

**Objective:** The CI `actions/cache` step uses `continue-on-error: true` which
means cache misses/restore failures are silently dropped. Also, the cache keys
don't include the compiler version, causing full rebuilds when the CI image
updates.

**Files:**
- Modify: `.github/workflows/ci.yml`

**Changes:**
1. Remove `continue-on-error: true` from ccache steps (or keep it but log)
2. Add compiler version to cache key: `ccache-${{ hashFiles('docker/Dockerfile.ci') }}-${{ github.sha }}`
3. Use `restore-keys:` that fall back to the latest cache from the same CI image build

**Verification:** CI builds get cache hits across pushes on the same branch.

## Task 8: /proc/uptime

**Objective:** Expose system uptime via procfs so any process can read it.
Currently uptime is only available via the `uptime` shell command.

**Files:**
- Modify: `src/fs/procfs.c`
- Modify: `src/include/timer.h` (export timer_get_ticks if not already)

**Implementation:** Add a `procfs_uptime()` function that formats:

```
uptime: 12345.67 seconds
```

Format uses `timer_get_ticks() / TIMER_FREQ` for whole seconds and
`timer_get_ticks() % TIMER_FREQ * 100 / TIMER_FREQ` for centiseconds.

**Verification:** `cat /proc/uptime` displays uptime.

## Task 10: LTO and dead-code elimination

**Objective:** Reduce kernel binary size and improve optimization.

**Files:**
- Modify: `Makefile`

**Changes:**
```makefile
CFLAGS += -flto -fdata-sections -ffunction-sections
LDFLAGS += -Wl,--gc-sections
```

Wrap in a conditional so it can be disabled for debugging:
```makefile

CFLAGS += -flto -fdata-sections -ffunction-sections
LDFLAGS += -Wl,--gc-sections
endif
```

**Verification:** `make && ls -lh build/kernel.bin` shows reduced size.
`make NO_LTO=1` builds without LTO for faster debug builds.
